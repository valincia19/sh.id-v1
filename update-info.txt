Major update introducing multi-layer script protection, universal executor compatibility for mobile and desktop, and deployment experience improvements.

- **Protection Quota**: Monthly usage tracking system with real-time display of remaining and total limits.

- **Fresh Protection Support**: Ability to generate new variants from the original script at any time.

- **Cross-Platform Support**: Automatic compatibility layer for executors lacking built-in libraries, primarily targeting mobile stability.
- **Safe Execution Wrapper**: Implemented multi-method fallbacks for restricted execution environments.
- **Smart Identification**: Enhanced device identification using multiple HWID detection methods for better reliability.
- **Timestamp Precision**: Deployment timestamps now display exact date and time.

- **Challenge/Verify System**: Mandatory device verification before script delivery.
- **Encoded Injection**: Endpoint and key protection via computationally hidden variables.
- **Environment Detection**: Restricted access allowing only verified executors to load protected content.
- **Server-Side Injection**: Automatic compatibility updates without requiring manual script changes.
- **Performance Caching**: Request caching implemented to handle traffic spikes and ensure low latency.
- **Automated Tracking**: Integrated usage tracking for advanced analytics and stability monitoring.

**[FIXES]**
- **Mobile Stabilization**: Resolved execution crashes on mobile executors caused by missing standard libraries.
- **Math Fallback**: Fixed decryption failures when standard math libraries are unavailable by implementing an internal fallback.
